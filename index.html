SIMPHP Framework

O SIMPHP é um mini framework desenvolvido em PHP OO com o padrão MVC para agilizar o processo de criação de aplicações PHP OO. Possui uma estrutura simples e intuitiva e já vem com algumas classes ajudantes (HELPERS) para gerar tags HTML, fazer uploads e downloads de arquivos, autenticação de usuários, redirecionar páginas, enviar e-mails (utilizando a biblioteca PHPMailer), ler e gravar parâmetros em arquivos .ini entre outros. Para usar o SIMPHP basta copiar todo o conteúdo do framework para o diretório do servidor onde a sua aplicação irá rodar e começar a desenvolver. Existe um parâmetro no arquivo index.php que deve ser configurado antes de começar a executar sua aplicação no servidor. A constante PATH_ROOT deve receber o caminho do diretório onde a sua aplicação está alocada dentro do servidor. Se você utiliza o apache e sua aplicação está dentro de um diretório chamado “meu_site” no diretório raiz do seu servidor a constante PATH_ROOT deve ser configurada como define(‘PATH_ROOT’, ‘meu_site’).

Se você estiver usando o IIS, deverá levar em consideração dois cenários: • Você criou um novo site no Gerenciador do IIS. Neste caso o seu site será executado no diretório raiz do seu site virtual, logo a constante PATH_ROOT deverá ser definida com nenhum valor.

Ex.: define('PATH_ROOT', DS . 'appname' . DS);

Obs.: definir o valor de PATH_ROOT com NULL não é recomendado.

• Você criou um novo aplicativo dentro de um site virtual existente no gerenciador do IIS.

Neste caso a constante PATH_ROOT deve ser configurada com o mesmo valor definido no campo alias ao criar o aplicativo. define('PATH_ROOT', DS . 'meu_site' . DS);

Estrutura de diretórios

O SIMPHP possui uma estrutura de diretórios simples porem, rígida. Por seguir o padrão MVC, cada tipo de arquivo deve ser criado de uma forma específica e em um local especifico.

Toda a lógica e acesso à dados da aplicação está separada do HMTL. Fazemos o acesso a dados e manipulação do banco nas classes Models dentro de app>models. A lógica e regra de negócio da nossa aplicação estará dentro das classes controllers em add>controllers e todo o HTML e geração de visualizações de páginas será colocado dentro das views em app>views.

Os arquivos estáticos da nossa aplicação (CSS, JS, Imagens) deverão ser colocadas dentro de seus respectivos diretórios em public->templates>nome_template->css, public->templates>nome_template->js ou public->templates>nome_template->img. Você pode criar quantos templates achar necessário.

Padrão de Nomenclatura de arquivos

No SIMPHP seguimos o seguinte padrão para nomear nossos arquivos.

Todos os arquivos deverão ser nomeados com a primeira letra do nome do arquivo em maiúscula. Caso o nome seja composto, a letra inicial de cada palavra no nome do arquivo também deverá estar em maiúscula. Ficam excluídos dessa regra os seguintes tipos de arquivo:

  • CSS
  • Imagens
  • JavaScripts
  • Bibliotecas de terceiros.
  • Áudio

  • Vídeo.

    Ex.:

    Controller -> IndexController.php

    Model -> IndexModel.php

    View -> Index.phtml

Os arquivos de views devem obrigatoriamente ter a extensão .phtml para serem reconhecidos pelo SIMPHP.

Acesso a banco de dados

O acesso a banco de dados está definido na classe Model em root>system>model.php e está pré-configurado para acessar bases de dados via PDO. Altere a classe model de acordo com sua preferência.
As configurações de acesso à banco de dados (usuário, senha, servidor) se encontra no arquivo de configuração em root>system>config>config.ini. O mesmo arquivo também mantém as configurações de SMTP utilizadas pelo EmailHelper para envio de e-mails.

Exibindo ou Ocultando Erros

Para exibir ou ocultar os erros gerados pelo PHP basta alterar o valor da constante DEVELOPMENT_ENVIRONMENT, no arquivo index.php no diretório raiz da aplicação. Ocultando erros define('DEVELOPMENT_ENVIRONMENT', false); Habilitando erros define('DEVELOPMENT_ENVIRONMENT', true);
